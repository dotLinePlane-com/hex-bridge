# HEX-Bridge 功能设计文档

> 本目录存放各功能模块的**功能描述文档**，说明每个模块做什么、依赖什么硬件、涉及哪些协议命令码。
> 命令码详情见 `../protocol/` 目录，实现设计见 `../firmware/` 目录，测试用例见 `../test/` 目录。

## 文档内容规范

每个功能文档应包含：

1. **功能描述** — 一句话说明该模块的作用
2. **子功能** — 表格列出子功能和命令码范围
3. **硬件依赖** — GPIO、芯片、关键约束
4. **设计文档索引** — 指向协议定义、固件设计、测试用例的路径

> 功能文档不包含命令详情、数据结构、实现流程等——这些分别属于协议文档和固件设计文档的职责。
