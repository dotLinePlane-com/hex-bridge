## 架构概览

```
┌─────────────────────────────────────────────────────┐
│                   app_main()                         │
│  NVS Init → MsgBus Init → Transport Init → Modules  │
└─────────────────────────────────────────────────────┘

┌────────────┐  ┌────────────┐  ┌────────────┐
│ MCP 通信层  │  │ UART 模块   │  │ System 模块│  ...
│ (UART1)    │  │ (UART2)    │  │            │
│ recv_task  │  │ rx_task    │  │            │
│ send_task  │  │            │  │            │
└─────┬──────┘  └─────┬──────┘  └─────┬──────┘
      │               │               │
      └───────────────┴───────┬───────┘
                              │
                    ┌─────────▼─────────┐
                    │    消息总线         │
                    │  命令路由 + 事件聚合 │
                    └───────────────────┘
```

## 设计原则

1. **模块独立**：每个外设模块在独立的 FreeRTOS 任务中运行，通过消息总线通信
2. **零耦合**：模块之间不直接调用，全部通过 `msg_bus` 路由
3. **统一接口**：所有模块实现 `hex_module_t` 接口（init / handle_cmd / stop）
4. **流式解析**：MCP 接收采用在线反转义 + CRC，不需要双倍缓冲区
5. **可配置**：所有硬件参数在 `hex_config.h` 中集中定义
