# 13. 批量传输模式 (0x90-0x9F)

当需要连续传输大量数据时（如读取整个 SPI Flash），使用批量模式减少帧头开销。

## 命令码一览

| 命令码 | 名称 | 方向 | 说明 |
|:---|:---|:---|:---|
| `0x90` | BULK_START | 请求-响应 | 开始批量传输 |
| `0x91` | BULK_DATA | 双向 | 批量数据帧 |
| `0x92` | BULK_ACK | 双向 | 批量传输确认 |
| `0x93` | BULK_STOP | 请求-响应 | 停止批量传输 |
| `0x94-0x9F` | — | — | 保留 |

> **v1.0 修正**：新增 BULK_ACK 命令，解决了原协议中批量传输无校验无重传的问题。

---

## 13.1 BULK_START (0x90) — 开始批量传输

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | TargetModule | u8 | 目标接口类型 |
| 1 | TargetChannel | u8 | 目标通道号 |
| 2-5 | TotalFrames | u32 | 总帧数 (升级为 u32 以支持 >65535 帧的大容量传输) |
| 6-7 | FrameSize | u16 | 每帧载荷大小（固定长度，最后一帧可以更短） |
| 8 | Direction | u8 | 传输方向 |
| 9 | WindowSize | u8 | 滑动窗口大小（发送多少帧后等待 ACK，默认 8） |

### TargetModule 定义

| 值 | 模块 |
|:---|:---|
| 0x10 | CAN |
| 0x20 | SPI |
| 0x30 | I2C |
| 0xA0 | UART |

### Direction 定义

| 值 | 方向 |
|:---|:---|
| 0x00 | 主机 → 设备 (下载) |
| 0x01 | 设备 → 主机 (上传) |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1-4 | SessionID | u32 | 批量会话 ID |
| 5 | ActualWindow | u8 | 实际窗口大小（设备可能调整） |

---

## 13.2 BULK_DATA (0x91) — 批量数据帧

### 请求/响应 (双向通用)

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-3 | SessionID | u32 | 会话 ID |
| 4-7 | FrameIndex | u32 | 帧序号（从 0 开始，升级为 u32） |
| 8-9 | DataCRC | u16 | 本帧数据的 CRC16 校验 |
| 10... | Data | u8[FrameSize] | 载荷数据 |

> **v1.0 修正**：每帧新增独立的 DataCRC 字段，不再依赖"上层保证完整性"。CRC 算法与全局帧相同 (CRC-16/CCITT-FALSE)，计算范围为 Data 部分。

---

## 13.3 BULK_ACK (0x92) — 批量传输确认

接收方每收到 WindowSize 帧数据后发送一次 ACK。

### 确认帧

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-3 | SessionID | u32 | 会话 ID |
| 4-7 | AckFrameIndex | u32 | 确认到的帧序号（含此序号及之前的帧，升级为 u32） |
| 8 | AckStatus | u8 | 确认状态 |
| 9-12 | RetransmitIndex | u32 | 需要重传的起始帧序号（仅 AckStatus=0x01 时有效，升级为 u32） |

### AckStatus 定义

| 值 | 状态 | 说明 |
|:---|:---|:---|
| 0x00 | OK | 所有帧正确接收 |
| 0x01 | NACK | 从 RetransmitIndex 开始需要重传 |
| 0x02 | ABORT | 接收方要求中止传输 |

### 重传机制

1. 发送方发送 WindowSize 帧后等待 ACK
2. 收到 OK → 继续发送下一窗口
3. 收到 NACK → 从 RetransmitIndex 重新发送
4. ACK 超时 (默认 1000ms) → 重传整个窗口
5. 连续 3 次重传失败 → 中止传输

---

## 13.4 BULK_STOP (0x93) — 停止批量传输

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-3 | SessionID | u32 | 会话 ID |
| 4 | Reason | u8 | 停止原因 |

### Reason 定义

| 值 | 原因 |
|:---|:---|
| 0x00 | 正常完成 |
| 0x01 | 用户取消 |
| 0x02 | 传输错误 |
| 0x03 | 超时 |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1-4 | SuccessFrames | u32 | 成功传输的帧数 (升级为 u32) |
| 5-8 | FailedFrames | u32 | 失败/丢失的帧数 (升级为 u32) |
| 9-12 | TotalBytes | u32 | 总传输字节数 |

---

## 批量传输流程

```
主机                                        设备
  │                                          │
  │  BULK_START(SPI, 100帧, 512B/帧, 窗口=8)  │
  ├─────────────────────────────────────────→│
  │                                          │
  │  BULK_START 响应(SessionID=0x1234)        │
  │←─────────────────────────────────────────┤
  │                                          │
  │  BULK_DATA(帧0) ──→                      │
  │  BULK_DATA(帧1) ──→                      │
  │  ...                                     │
  │  BULK_DATA(帧7) ──→                      │ 发满一个窗口
  │                                          │
  │  BULK_ACK(确认到帧7, OK)                  │
  │←─────────────────────────────────────────┤
  │                                          │
  │  BULK_DATA(帧8-15) ──→                   │ 下一个窗口
  │                                          │
  │  BULK_ACK(帧13, NACK, 重传从帧14)         │ 帧14 CRC错误
  │←─────────────────────────────────────────┤
  │                                          │
  │  BULK_DATA(帧14-15) ──→                  │ 重传
  │                                          │
  │  BULK_ACK(帧15, OK)                      │
  │←─────────────────────────────────────────┤
  │                                          │
  │  ... 继续直到帧99 ...                     │
  │                                          │
  │  BULK_STOP(正常完成)                      │
  ├─────────────────────────────────────────→│
  │                                          │
  │  BULK_STOP 响应(成功100帧, 失败0帧)       │
  │←─────────────────────────────────────────┤
```
