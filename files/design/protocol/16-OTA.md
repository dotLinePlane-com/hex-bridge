# 16. OTA 固件升级命令 (0xB0-0xBF)

支持通过通信协议对设备进行固件 OTA (Over-The-Air) 升级。基于 ESP-IDF 的双分区 OTA 机制设计。

## 命令码一览

| 命令码 | 名称 | 方向 | 说明 |
|:---|:---|:---|:---|
| `0xB0` | OTA_BEGIN | 请求-响应 | 开始 OTA 升级 |
| `0xB1` | OTA_DATA | 请求-响应 | 发送固件数据块 |
| `0xB2` | OTA_END | 请求-响应 | 结束 OTA 升级并验证 |
| `0xB3` | OTA_STATUS | 请求-响应 | 查询 OTA 进度 |
| `0xB4` | OTA_ROLLBACK | 请求-响应 | 回滚到上一版本固件 |
| `0xB5` | OTA_GET_PARTITION | 请求-响应 | 获取分区信息 |
| `0xB6` | OTA_PROGRESS | 事件上报 | OTA 进度上报 |
| `0xB7-0xBF` | — | — | 保留 |

---

## 16.1 OTA_BEGIN (0xB0) — 开始 OTA 升级

发起 OTA 升级会话。设备收到后准备接收固件数据（擦除目标分区等）。

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-3 | FirmwareSize | u32 | 固件总大小（字节） |
| 4-6 | FwVersion | u8[3] | 新固件版本（主.次.补丁） |
| 7-8 | ChunkSize | u16 | 每个数据块的大小（建议 1024 或 2048，固定最大 2048 限制） |
| 9-40 | SHA256 | u8[32] | 固件的 SHA-256 校验和 |
| 41 | Flags | u8 | OTA 标志 |

### Flags 定义

| Bit | 名称 | 说明 |
|:---|:---|:---|
| 0 | ForceUpdate | 1 = 强制升级（即使版本相同或更低） |
| 1 | AutoReboot | 1 = 验证成功后自动重启（默认） |
| 2 | ReportProgress | 1 = 每块数据都上报进度事件 |
| 3-7 | — | 保留 |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1-4 | SessionID | u32 | OTA 会话 ID |
| 5-6 | ActualChunkSize | u16 | 设备接受的实际块大小（可能调整） |
| 7-10 | FreeSpace | u32 | 目标分区可用空间（字节） |
| 11-13 | CurrentFwVer | u8[3] | 当前运行的固件版本 |

---

## 16.2 OTA_DATA (0xB1) — 发送固件数据块

按顺序发送固件二进制数据。

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-3 | SessionID | u32 | OTA 会话 ID |
| 4-5 | ChunkIndex | u16 | 数据块序号（从 0 开始） |
| 6-7 | ChunkCRC | u16 | 本帧 Data 数据块的 CRC16 校验值 (算法同全局帧，计算范围为 Data 部分) |
| 8-9 | DataLen | u16 | 数据长度 (N) |
| 10... | Data | u8[N] | 固件数据 |

> **注意**：最后一个数据块的 DataLen 可以小于 ChunkSize。

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1-2 | AckChunkIndex | u16 | 确认的数据块序号 |
| 3-6 | BytesWritten | u32 | 累计已写入字节数 |

### 块级校验与重传机制

1. **块级校验流程**：设备接收到 `OTA_DATA` 后，首先对载荷中的 `Data` 执行 `ChunkCRC` 校验。
   * 如果校验失败，设备响应 `Status = ERR_CRC (0x08)`，且 `AckChunkIndex` 保持为上一次成功写入的块序号，不递增 `BytesWritten`。
   * 主机收到 `ERR_CRC` 后，应当立即重新发送当前块。
2. **常规重传规则**：
   * 如果响应中 `AckChunkIndex` 小于发送的 `ChunkIndex`，说明中间有数据块丢失，主机需从 `AckChunkIndex + 1` 开始重传。
   * 设备连续收到相同 `ChunkIndex` 的数据块会自动去重。
   * 超时未收到响应（默认 3 秒），主机应重传当前块。
   * 连续 5 次重传失败，主机应调用 `OTA_END` 并设置 Action=0x02 中止整个升级过程。

## 16.3 OTA_END (0xB2) — 结束 OTA 升级

通知设备所有数据已发送完毕，设备执行完整性校验。

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-3 | SessionID | u32 | OTA 会话 ID |
| 4 | Action | u8 | 结束动作 |

### Action 定义

| 值 | 动作 | 说明 |
|:---|:---|:---|
| 0x00 | 验证并应用 | 校验 SHA-256，成功则设置新固件为启动分区 |
| 0x01 | 仅验证 | 校验但不切换启动分区 |
| 0x02 | 中止 | 丢弃已接收的数据，取消本次 OTA |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1 | VerifyResult | u8 | 校验结果 |
| 2-5 | TotalBytesWritten | u32 | 总写入字节数 |
| 6-9 | ElapsedMs | u32 | OTA 总耗时（毫秒） |

### VerifyResult 定义

| 值 | 结果 | 说明 |
|:---|:---|:---|
| 0x00 | 校验成功 | SHA-256 匹配，固件有效 |
| 0x01 | SHA-256 不匹配 | 数据完整性校验失败 |
| 0x02 | 大小不匹配 | 实际接收大小与声明不符 |
| 0x03 | 固件格式无效 | 固件头部校验失败（不是合法的 ESP-IDF 固件） |
| 0x04 | 已中止 | Action=0x02 时返回 |

> **注意**：如果 OTA_BEGIN 中 AutoReboot=1 且校验成功，设备将在发送响应后 2 秒自动重启。主机应做好重新连接的准备。

## 16.4 OTA_STATUS (0xB3) — 查询 OTA 进度

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-3 | SessionID | u32 | OTA 会话 ID (0x00000000 = 查询当前会话) |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1 | OtaState | u8 | 当前 OTA 状态 |
| 2-3 | ChunksReceived | u16 | 已接收的数据块数 |
| 4-5 | ChunksTotal | u16 | 总数据块数 |
| 6-9 | BytesReceived | u32 | 已接收字节数 |
| 10-13 | BytesTotal | u32 | 总字节数 |
| 14 | ProgressPercent | u8 | 进度百分比 (0-100) |

### OtaState 定义

| 值 | 状态 | 说明 |
|:---|:---|:---|
| 0x00 | IDLE | 空闲，无 OTA 进行中 |
| 0x01 | PREPARING | 准备中（擦除分区） |
| 0x02 | RECEIVING | 正在接收数据 |
| 0x03 | VERIFYING | 正在校验固件 |
| 0x04 | APPLYING | 正在应用（切换启动分区） |
| 0x05 | REBOOTING | 即将重启 |
| 0x06 | ABORTED | 已中止 |
| 0x07 | FAILED | 失败 |

---

## 16.5 OTA_ROLLBACK (0xB4) — 回滚到上一版本固件

当新固件运行异常时，回滚到之前的固件。

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | RollbackType | u8 | 回滚类型 |

### RollbackType 定义

| 值 | 类型 | 说明 |
|:---|:---|:---|
| 0x00 | 回滚并重启 | 切换到上一分区并立即重启 |
| 0x01 | 仅标记回滚 | 标记回滚，下次重启时生效 |
| 0x02 | 确认当前固件 | 确认当前固件有效（取消自动回滚） |

> **自动回滚机制**：新固件首次启动后，如果未在 60 秒内收到 `OTA_ROLLBACK(0x02)` 确认命令，设备将在下次重启时自动回滚到旧固件。

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1 | BootPartition | u8 | 回滚后的启动分区编号 |

---

## 16.6 OTA_GET_PARTITION (0xB5) — 获取分区信息

### 请求

载荷为空。

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1 | PartitionCount | u8 | OTA 分区数量 (N, 通常为 2) |
| 2 | ActivePartition | u8 | 当前运行的分区编号 |
| 3 | NextPartition | u8 | 下次 OTA 的目标分区编号 |
| 4 | FwConfirmed | u8 | 当前固件是否已确认 (0=未确认, 1=已确认) |
| 5... | Partitions | — | N 个分区信息 (每个 16 字节) |

### 每个分区信息结构 (16 字节)

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Index | u8 | 分区编号 |
| 1 | State | u8 | 分区状态 |
| 2-4 | FwVersion | u8[3] | 固件版本 (无固件时为 0x00 0x00 0x00) |
| 5-8 | FwSize | u32 | 固件大小 (字节) |
| 9-12 | PartSize | u32 | 分区总大小 (字节) |
| 13-15 | — | u8[3] | 保留 |

### 分区 State 定义

| 值 | 状态 | 说明 |
|:---|:---|:---|
| 0x00 | 空 | 分区无有效固件 |
| 0x01 | 有效 | 包含有效固件 |
| 0x02 | 运行中 | 当前正在运行的固件 |
| 0x03 | 待验证 | 新固件等待确认 |
| 0x04 | 无效 | 固件校验失败 |

---

## 16.7 OTA_PROGRESS (0xB6) — OTA 进度上报 (事件上报)

当 OTA_BEGIN 中 ReportProgress=1 时，设备在写入每块数据后主动上报进度。

### 设备 → 主机主动上报

Flags: DIR=1, EVT=1

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-3 | SessionID | u32 | OTA 会话 ID |
| 4 | OtaState | u8 | 当前状态 (同 16.4 节) |
| 5 | ProgressPercent | u8 | 进度百分比 (0-100) |
| 6-9 | BytesReceived | u32 | 已接收字节数 |

---

## OTA 升级流程

```
主机                                          设备
  │                                            │
  │  ① OTA_GET_PARTITION()                     │
  ├───────────────────────────────────────────→│  查询当前分区状态
  │  (分区信息: ota_0 运行中, ota_1 空)          │
  │←───────────────────────────────────────────┤
  │                                            │
  │  ② OTA_BEGIN(大小, 版本, SHA256, 块大小)     │
  ├───────────────────────────────────────────→│  设备擦除 ota_1 分区
  │  (SessionID=0x5678, 空间足够)               │
  │←───────────────────────────────────────────┤
  │                                            │
  │  ③ OTA_DATA(块0, 2048B)                    │
  ├───────────────────────────────────────────→│
  │  (ACK 块0, 已写入 2048B)                    │
  │←───────────────────────────────────────────┤
  │                                            │
  │  ④ OTA_DATA(块1, 2048B)                    │
  ├───────────────────────────────────────────→│
  │  (ACK 块1, 已写入 4096B)                    │
  │←───────────────────────────────────────────┤
  │                                            │
  │  ← OTA_PROGRESS(5%, 8192B)                 │  进度上报 (可选)
  │←───────────────────────────────────────────┤
  │                                            │
  │  ... 继续发送直到全部完成 ...                 │
  │                                            │
  │  ⑤ OTA_END(验证并应用)                      │
  ├───────────────────────────────────────────→│  设备校验 SHA-256
  │  (校验成功, 即将重启)                        │
  │←───────────────────────────────────────────┤
  │                                            │
  │  ~~~~ 设备重启 (2秒后) ~~~~                  │
  │                                            │
  │  ⑥ PING()  (重新连接)                       │
  ├───────────────────────────────────────────→│
  │  (设备在线, 新版本运行中)                     │
  │←───────────────────────────────────────────┤
  │                                            │
  │  ⑦ OTA_ROLLBACK(确认当前固件)               │
  ├───────────────────────────────────────────→│  确认新固件有效
  │  (确认成功)                                  │
  │←───────────────────────────────────────────┤
```

### 异常处理

| 场景 | 处理方式 |
|:---|:---|
| 传输中断 | 主机可重发 OTA_BEGIN 重新开始，或 OTA_END(中止) 放弃 |
| SHA-256 校验失败 | OTA_END 返回 VerifyResult=0x01，分区不切换 |
| 空间不足 | OTA_BEGIN 响应 Status=ERR_OTA_NO_SPACE |
| 新固件启动失败 | 设备看门狗触发重启，自动回滚到旧分区 |
| 确认超时 | 60 秒内未确认，下次重启自动回滚 |
| 数据块乱序 | 设备拒绝乱序块，返回期望的 ChunkIndex |
