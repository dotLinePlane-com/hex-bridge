# 7. UART 接口命令 (0xA0-0xAF)

## 命令码一览

| 命令码 | 名称 | 方向 | 说明 |
|:---|:---|:---|:---|
| `0xA0` | UART_OPEN | 请求-响应 | 打开 UART 通道 |
| `0xA1` | UART_CLOSE | 请求-响应 | 关闭 UART 通道 |
| `0xA2` | UART_CONFIG | 请求-响应 | 配置 UART 参数 |
| `0xA3` | UART_SEND | 请求-响应 | 发送 UART 数据 |
| `0xA4` | UART_RECV | 事件上报 | 接收 UART 数据 |
| `0xA5` | UART_SET_BREAK | 请求-响应 | 发送 Break 信号 |
| `0xA6` | UART_STATUS | 请求-响应 | 获取 UART 状态 |
| `0xA7` | UART_FLUSH | 请求-响应 | 清空 UART 缓冲区 |
| `0xA8-0xAF` | — | — | 保留 |

---

## 7.1 UART_OPEN (0xA0) — 打开 UART 通道

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | RxMode | u8 | 接收模式 |

### RxMode 定义

| 值 | 模式 | 说明 |
|:---|:---|:---|
| 0x00 | 被动上报 | 收到数据自动上报 (默认) |
| 0x01 | 行模式 | 按行分割上报 (遇到 `\n` 或 `\r\n` 上报) |
| 0x02 | 定长模式 | 累积到指定长度后上报 |
| 0x03 | 超时模式 | 收到首字节后超时未再收到新字节则上报 |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1-2 | RxBufSize | u16 | 接收缓冲区大小（字节） |
| 3-4 | TxBufSize | u16 | 发送缓冲区大小（字节） |

---

## 7.2 UART_CLOSE (0xA1) — 关闭 UART 通道

### 请求

载荷为空。

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |

---

## 7.3 UART_CONFIG (0xA2) — 配置 UART 参数

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-3 | BaudRate | u32 | 波特率 (如 9600, 115200, 921600) |
| 4 | DataBits | u8 | 数据位 |
| 5 | StopBits | u8 | 停止位 |
| 6 | Parity | u8 | 校验位 |
| 7 | FlowControl | u8 | 流控模式 |
| 8-9 | RxThreshold | u16 | 接收上报阈值 (与 RxMode 配合) |
| 10 | RxTimeout | u8 | 接收超时 (毫秒, 用于超时模式) |

### DataBits 定义

| 值 | 含义 |
|:---|:---|
| 0x05 | 5 位数据 |
| 0x06 | 6 位数据 |
| 0x07 | 7 位数据 |
| 0x08 | 8 位数据 (默认) |

### StopBits 定义

| 值 | 含义 |
|:---|:---|
| 0x01 | 1 位停止位 (默认) |
| 0x02 | 1.5 位停止位 |
| 0x03 | 2 位停止位 |

### Parity 定义

| 值 | 含义 |
|:---|:---|
| 0x00 | 无校验 (默认) |
| 0x01 | 奇校验 (Odd) |
| 0x02 | 偶校验 (Even) |
| 0x03 | 固定为 1 (Mark) |
| 0x04 | 固定为 0 (Space) |

### FlowControl 定义

| 值 | 含义 |
|:---|:---|
| 0x00 | 无流控 (默认) |
| 0x01 | 硬件流控 (RTS/CTS) |
| 0x02 | 软件流控 (XON/XOFF) |

### RxThreshold 含义

| RxMode | RxThreshold 含义 |
|:---|:---|
| 被动上报 (0x00) | 缓冲区累积超过此字节数时上报 (默认 1) |
| 行模式 (0x01) | 忽略 |
| 定长模式 (0x02) | 每次上报的固定字节数 |
| 超时模式 (0x03) | 忽略（由 RxTimeout 控制） |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1-4 | ActualBaud | u32 | 实际波特率 (可能因分频而略有偏差) |

---

## 7.4 UART_SEND (0xA3) — 发送 UART 数据

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-1 | DataLen | u16 | 数据长度 (N) |
| 2... | Data | u8[N] | 发送数据 |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1-2 | ActualLen | u16 | 实际写入发送缓冲区的字节数 |

> **注意**：ActualLen 表示数据已被写入设备发送缓冲区，并不代表已在物理线路上发送完成。

---

## 7.5 UART_RECV (0xA4) — 接收 UART 数据 (事件上报)

### 设备 → 主机主动上报

Flags: DIR=1, EVT=1

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | RxFlags | u8 | 接收状态标志 |
| 1-2 | DataLen | u16 | 数据长度 (N) |
| 3... | Data | u8[N] | 接收数据 |

### RxFlags 定义

| Bit | 名称 | 说明 |
|:---|:---|:---|
| 0 | BufferOverflow | 1 = 接收缓冲区曾发生溢出 |
| 1 | ParityError | 1 = 检测到校验错误 |
| 2 | FrameError | 1 = 检测到帧错误 |
| 3 | BreakDetect | 1 = 检测到 Break 信号 |
| 4-7 | — | 保留 |

---

## 7.6 UART_SET_BREAK (0xA5) — 发送 Break 信号

Break 信号是将 TX 线拉低一段时间（通常 > 1 个字符帧的时间）。

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-1 | DurationMs | u16 | Break 持续时间 (毫秒, 0 = 默认 10ms) |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |

---

## 7.7 UART_STATUS (0xA6) — 获取 UART 状态

### 请求

载荷为空。

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1-4 | BaudRate | u32 | 当前波特率 |
| 5 | LineState | u8 | 线路状态 |
| 6-7 | TxBufUsed | u16 | 发送缓冲区已用字节数 |
| 8-9 | RxBufUsed | u16 | 接收缓冲区已用字节数 |
| 10-13 | TxCount | u32 | 总发送字节数 |
| 14-17 | RxCount | u32 | 总接收字节数 |
| 18 | ErrorCount | u8 | 错误计数 (校验错误 + 帧错误) |

### LineState 定义

| Bit | 名称 | 说明 |
|:---|:---|:---|
| 0 | TxIdle | 1 = 发送空闲 |
| 1 | RxActive | 1 = 正在接收数据 |
| 2 | CTS | CTS 引脚状态 (仅硬件流控时有效) |
| 3 | DSR | DSR 引脚状态 (如果可用) |
| 4 | BreakActive | 1 = Break 信号正在发送 |
| 5-7 | — | 保留 |

---

## 7.8 UART_FLUSH (0xA7) — 清空 UART 缓冲区

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | FlushType | u8 | 清空类型 |

### FlushType 定义

| 值 | 含义 |
|:---|:---|
| 0x00 | 清空接收缓冲区 |
| 0x01 | 清空发送缓冲区 |
| 0x02 | 清空所有缓冲区 |
| 0x03 | 等待发送完成后清空 (Drain) |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |

---

## 典型使用流程

```
1. UART_OPEN(RxMode=行模式)          ← 打开通道，按行接收
2. UART_CONFIG(115200, 8N1, 无流控)   ← 配置参数
3. UART_SEND("AT\r\n")               ← 发送 AT 命令
4. ← UART_RECV("OK\r\n")             ← 设备上报收到的响应
5. UART_STATUS()                      ← 查看状态 (可选)
6. UART_CLOSE()                       ← 关闭通道
```
