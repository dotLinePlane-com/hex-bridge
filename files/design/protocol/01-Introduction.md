# 1. 协议概述

## 1.1 设计目标

| 目标 | 描述 |
|:---|:---|
| **统一帧结构** | 所有接口使用相同的帧头格式，降低解析复杂度 |
| **高效传输** | 10 字节固定头部 + 变长载荷，最小化开销 |
| **双向异步** | 支持请求-响应模型和设备主动事件上报 |
| **事务匹配** | 2 字节序列号支持请求-响应精确配对 |
| **多通道支持** | 同一设备可管理多个接口通道（最多 256 个） |
| **可扩展性** | 预留充足的命令码空间，支持未来扩展 |
| **可靠性** | CRC-16/CCITT-FALSE 校验 + 0x7E 帧尾 + 字节转义 |
| **流量控制** | 内置流控机制，防止缓冲区溢出 |

## 1.2 全局约定

### 1.2.1 字节序

> **本协议所有多字节字段均使用大端序（Big-Endian / Network Byte Order）。**

即高位字节在前，低位字节在后。例如 `0x1234` 在线路上依次传输 `0x12, 0x34`。

### 1.2.2 数据类型

| 标记 | 含义 |
|:---|:---|
| `u8` | 无符号 8 位整数 |
| `u16` | 无符号 16 位整数，大端序 |
| `u32` | 无符号 32 位整数，大端序 |
| `u8[N]` | N 字节的字节数组 |
| `str(L)` | L 字节的 ASCII 字符串（不含终止符） |

### 1.2.3 时间戳约定

所有时间戳字段统一使用 **微秒（μs）** 为单位，32 位无符号整数，表示设备启动以来的时间。溢出周期约 71.6 分钟，溢出后从 0 重新计数。

## 1.3 通信模型

```
   主机 (MCP Server)                    设备 (HEX-Bridge 固件)
        │                                      │
        │  ① 请求 (Request)  [DIR=0, ACK=1]    │
        ├─────────────────────────────────────→ │
        │                                      │
        │  ② 响应 (Response) [DIR=1, SeqNum匹配]│
        │ ←─────────────────────────────────────┤
        │                                      │
        │  ③ 事件上报 (Event) [DIR=1, EVT=1]   │
        │ ←─────────────────────────────────────┤
        │                                      │
        │  ④ 命令无需响应 (Fire-and-Forget)      │
        │     [DIR=0, ACK=0]                   │
        ├─────────────────────────────────────→ │
        │                                      │
        │  ⑤ 流控通知 [FLOW_CONTROL]            │
        │ ←─────────────────────────────────────┤
        │                                      │
```

### 1.3.1 请求-响应匹配规则

1. 主机发送请求时设置唯一的 `SeqNum`（序列号）
2. 设备响应时**必须**回填相同的 `SeqNum`
3. 事件上报的 `SeqNum` 由设备自行递增，与请求无关
4. 主机应维护超时机制，超时未收到匹配响应视为通信失败

### 1.3.2 传输层

本协议工作在 USB CDC 虚拟串口之上，物理传输参数：

| 参数 | 值 |
|:---|:---|
| 波特率 | USB CDC (与波特率无关) |
| 数据位 | 8 |
| 校验位 | 无 |
| 停止位 | 1 |

## 1.4 命令码空间分配

```
0x00-0x0F  ████████ 系统管理
0x10-0x1F  ████████ CAN 接口
0x20-0x2F  ████████ SPI 接口
0x30-0x3F  ████████ I2C 接口
0x40-0x4F  ████████ 网络配置
0x50-0x5F  ████████ TCP 协议
0x60-0x6F  ████████ UDP 协议
0x70-0x7F  ████████ WebSocket
0x80-0x8F  ████████ GPIO 接口
0x90-0x9F  ████████ 批量传输
0xA0-0xAF  ████████ UART 接口
0xB0-0xBF  ████████ OTA 固件升级
0xC0-0xFF  ░░░░░░░░ 保留 (未来扩展)
```

## 1.5 支持的接口

| 接口 | 通道数 | 说明 |
|:---|:---|:---|
| CAN / CAN FD | 1-2 | 支持经典 CAN 和 CAN FD (最大 64 字节) |
| SPI | 1-4 | 支持全双工、半双工、手动 CS 控制 |
| I2C | 1-2 | 支持 7 位和 10 位地址模式 |
| UART | 1-4 | 支持可配置波特率、数据位、校验、流控 |
| 以太网 TCP | 多连接 | Server/Client 模式 |
| 以太网 UDP | 多连接 | Server/Client 模式，支持广播/多播 |
| WebSocket | 多连接 | Server/Client 模式 |
| GPIO | 32 引脚 | 输入/输出、中断、批量操作 |
| OTA | — | 固件升级、双分区切换、回滚、SHA-256 校验 |
