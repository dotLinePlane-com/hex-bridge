# 12. GPIO 接口命令 (0x80-0x8F)

## 命令码一览

| 命令码 | 名称 | 方向 | 说明 |
|:---|:---|:---|:---|
| `0x80` | GPIO_SET_DIR | 请求-响应 | 设置 GPIO 方向 |
| `0x81` | GPIO_WRITE | 请求-响应 | 设置 GPIO 输出 |
| `0x82` | GPIO_READ | 请求-响应 | 读取 GPIO 输入 |
| `0x83` | GPIO_SET_PULL | 请求-响应 | 设置上下拉 |
| `0x84` | GPIO_INT_EN | 请求-响应 | 使能 GPIO 中断 |
| `0x85` | GPIO_INT_EVENT | 事件上报 | GPIO 中断事件 |
| `0x86` | GPIO_WRITE_MASK | 请求-响应 | 批量设置 GPIO 输出 |
| `0x87` | GPIO_READ_ALL | 请求-响应 | 批量读取 GPIO 状态 |
| `0x88-0x8F` | — | — | 保留 |

## 12.4 GPIO_SET_PULL (0x83) — 设置上下拉

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | PinNum | u8 | 引脚号 |
| 1 | PullMode | u8 | 上下拉模式 |

### PullMode 定义

| 值 | 模式 | 说明 |
|:---|:---|:---|
| 0x00 | 浮空 | 无上下拉 |
| 0x01 | 上拉 (Pull-Up) | 内部上拉电阻 |
| 0x02 | 下拉 (Pull-Down) | 内部下拉电阻 |
| 0x03 | 上下拉 (Pull-Up + Pull-Down) | 两者同时启用 |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |

---

## 12.5 GPIO_INT_EN (0x84) — 使能 GPIO 中断

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | PinNum | u8 | 引脚号 |
| 1 | TriggerMode | u8 | 中断触发条件 |
| 2 | DebounceMs | u8 | 中断消抖时间 (毫秒, 默认 10ms, 0 = 禁用中断) |

### TriggerMode 定义

| Bit | 名称 | 说明 |
|:---|:---|:---|
| 0 | RisingEdge | 上升沿触发 |
| 1 | FallingEdge | 下降沿触发 |
| 2 | HighLevel | 高电平触发 |
| 3 | LowLevel | 低电平触发 |
| 4-7 | — | 保留 |

> **注意**：TriggerMode = 0x03 (Bit 0 + Bit 1) 表示双边沿触发。DebounceMs = 0 且 TriggerMode = 0 时表示禁用该引脚的中断。

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |

## 12.6 GPIO_INT_EVENT (0x85) — GPIO 中断事件 (事件上报)

### 设备 → 主机主动上报

Flags: DIR=1, EVT=1, TS=1（建议启用时间戳）

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | PinNum | u8 | 触发中断的引脚号 |
| 1 | TriggerType | u8 | 实际触发类型 (同 TriggerMode 位定义) |
| 2 | CurrentLevel | u8 | 中断发生时的引脚电平 (0/1) |

> **v1.0 修正**：时间戳统一使用帧头的可选时间戳字段（微秒精度），而非载荷内单独的毫秒时间戳。

### ⚠️ 中断防抖与防风暴（Rate Limiting）规范
为了避免工业现场因为引脚异常抖动或连接高频脉冲信号产生“中断风暴”（这会导致串口缓冲区被占满、串口线程阻塞和 CPU 锁死），固件实现必须遵循以下规范：
1. **防风暴控制**：固件侧应实现单个 IO 中断上报的速率限制。推荐限制为：**单个引脚每秒最多允许上报 50 次 `GPIO_INT_EVENT` 中断事件**。一旦超过此阈值，固件应自动挂起该引脚中断（停止上报），并向主机上报错误或流控事件，直到下一秒计数器重置或主机重新使能。
2. **应用引导**：高频采集信号（频率 > 50Hz）禁止使用 `GPIO_INT_EVENT` 异步上报，而应引导用户采用定时器脉冲计数或批量模式。

## 12.7 GPIO_WRITE_MASK (0x86) — 批量设置 GPIO 输出

通过位掩码一次操作多个引脚。

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-3 | PinMask | u32 | 引脚掩码 (Bit N = 1 表示操作 Pin N) |
| 4-7 | PinValues | u32 | 引脚值 (Bit N = 1 表示 Pin N 输出高电平) |

### 示例

设置 Pin 0 = 高, Pin 1 = 低, Pin 5 = 高，不影响其他引脚：

```
PinMask  = 0x00000023  (Bit 0, 1, 5 = 1)
PinValues = 0x00000021  (Bit 0, 5 = 1, Bit 1 = 0)
```

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |

---

## 12.8 GPIO_READ_ALL (0x87) — 批量读取 GPIO 状态

一次读取所有引脚的当前状态。

### 请求

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0-3 | PinMask | u32 | 引脚掩码 (Bit N = 1 表示读取 Pin N, 0xFFFFFFFF = 读取全部) |

### 响应

| 偏移 | 字段 | 类型 | 说明 |
|:---|:---|:---|:---|
| 0 | Status | u8 | 状态码 |
| 1-4 | PinLevels | u32 | 引脚电平 (Bit N = Pin N 的电平) |
| 5-8 | PinDirs | u32 | 引脚方向 (Bit N = 1 表示 Pin N 为输出) |
